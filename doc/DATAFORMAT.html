DATAFORMAT
Author:
Matthias Rost (mrost@net.t-labs.tu-berlin.de)

This file describes the simple dataformat used for representing A-CVSAP instances.

Each file must start with a header containing the following information

NUMBER_OF_NODES <N>
NUMBER_OF_ARCS <N>
MIN_NUMBER_ACTIVE_RECEIVER_CONFIGURATIONS <N>
MAX_NUMBER_ACTIVE_RECEIVER_CONFIGURATIONS <N>
MAX_BUDGET_AGGREGATIOIN_INSTALLATION <N>
NAME_SSOURCE <S>
NAME_SSINK_AGGREGATION <S>
NAME_SSINK_RECEIVER <S>

where <N> must take an integral value and <S> is a string. Using the current implementation, you should set

MIN_NUMBER_ACTIVE_RECEIVER_CONFIGURATIONS 1
MAX_NUMBER_ACTIVE_RECEIVER_CONFIGURATIONS 1
MAX_BUDGET_AGGREGATIOIN_INSTALLATION M

where M is larger than the sum of costs over all Steiner sites.

Next you must specify the nodes line by line. For a simple node just write

N <NAME>

where <NAME> denotes the name.

For a sending node, i.e. a terminal, use

NS <NAME>

and for an aggregation node, i.e. a Steiner site, use

NA <NAME> <cap> <opening-cost> <usage-cost>

where <cap> is the integral (!) capacity, <opening-cost> is the cost for activating the node and <usage-cost> is an additionally incurred cost per connection. In our experiments, <usage-cost> was always set to 0.

Next you must specify the edges in the network. To do that, write them line by line in the following way:

E <TAIL-NAME> <HEAD-NAME> <cap> <cost>

where <TAIL-NAME> is the name of the node the edge leaves and <HEAD-NAME> is the name of the node the edge is directed to. <cap> must take an integral value and gives the capacity while <cost> is the real valued cost incurred for using this edge.

Lastly, you need to specify the root. Our framework generally allows for multiple receiver configurations. In our experiments we use only one configuration containing a single node. For specifying a single root, append

RCONF <CONF_NAME> 0.0
R <NAME> <cap>
RCONF_END

where <NAME> is the name of the root node and <cap> is its integral capacity, as we generally allow for multiple receiver configurations, <CONF_NAME> gives the name for the coniguration.
