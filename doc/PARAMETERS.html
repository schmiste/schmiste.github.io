PARAMETERS
Author:
Matthias Rost (mrost@net.t-labs.tu-berlin.de)

Our CVSAP solver allows for parameters to be passed via a file cvsap.set that must be contained in the working directory in which the executable bin/cvsap is executed.

The following parameters may be specified in addition to the parameters allowed by SCIP (see http://scip.zib.de/doc/html/PARAMETERS.shtml)

A.

constraints/Connectivity/scale = <N>
constraints/Connectivity/creep = [FALSE/TRUE]
constraints/Connectivity/max_number_of_cuts_before_lp_recalculation = <N>
constraints/Connectivity/nest_depth = <N>
constraints/Connectivity/threshold_of_absorbed_flow_to_start_aggregation_cuts = <R>
constraints/Connectivity/number_of_threads_for_flow_calculation = <N>

B.

aggregation_mip/integral_flow = [FALSE/TRUE]
aggregation_mip/debug/threading = [FALSE/TRUE]
aggregation_mip/debug/model_construction = [FALSE/TRUE]
aggregation_mip/debug/ConnectivityHandler = [FALSE/TRUE]
aggregation_mip/debug/connectivity_heuristic = [FALSE/TRUE]
aggregation_mip/debug/FlowDecoRound = [FALSE/TRUE]

C.

heuristics/FlowDecoRound/XYZ

Above <N> must be an integer while <R> may be a real valued number. We shortly explain the parameters below.

A.

constraints/Connectivity/scale
The scale parameter influences the flow calculations, as the real valued LP relaxation values are rounded in accordance to scale.

constraints/Connectivity/creep
Enables or disables the usage of creep flow.

constraints/Connectivity/max_number_of_cuts_before_lp_recalculation
Using this parameter, the search for further cuts is aborted if the number specified in this parameter is found.

constraints/Connectivity/nest_depth
This parameter controls the maximal depth of nested cuts.

constraints/Connectivity/threshold_of_absorbed_flow_to_start_aggregation_cuts
Using this parameter, connectivity inequalities for Steiner nodes are only checked if the absorbed amount of flow surpasses the value specified in this parameter (in case that the Steiner node is not fully activated).

constraints/Connectivity/number_of_threads_for_flow_calculation
Specifies the number of threads that shall be used for the flow computations.

B.

aggregation_mip/integral_flow
This parameter decides whether the flow variables must be integral or not. For our algorithm to work, this parameter must be set to TRUE!

We advise you to set the last 4 parameters to FALSE as they handle debug messages.

aggregation_mip/debug/threading
aggregation_mip/debug/model_construction
aggregation_mip/debug/ConnectivityHandler
aggregation_mip/debug/connectivity_heuristic
aggregation_mip/debug/FlowDecoRound

C.

We have implemented a preliminary heuristic for generating feasible solutions during the execution of the branch-and-cut process. All general SCIP heuristic parameters (see http://scip.zib.de/doc/html/HEUR.shtml) can be provided by using heuristics/FlowDecoRound/XYZ

To disable the heuristic use the following parameters:
heuristics/FlowDecoRound/freq = -1
heuristics/FlowDecoRound/freqofs = 0
heuristics/FlowDecoRound/maxdepth = 0
